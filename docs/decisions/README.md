# 의사결정 로그 (ADR — Architecture Decision Record)

더핀 인트라넷에서 **6개월 후 누군가 "왜 이렇게 짰지" 물을 만한 결정**을 여기 남깁니다.

## 어떤 결정을 기록하는가

- 멀티 페르소나 모드에서 충돌이 발생해 메인 Claude가 한쪽 입장을 택한 경우
- Google Sheets 스키마 변경 (컬럼 추가·제거·의미 변경)
- 외부 서비스(API·금융기관·법무 시스템) 연계 결정 또는 거절
- 7단계 워크플로우 변경 (단계 추가·제거·순서 변경)
- 빌드 도구·프레임워크 도입 검토 결과 (기각도 기록)
- RBAC 권한 매트릭스 변경
- 수수료 정책·분배 알고리즘 변경
- 폐기된 기능 — 왜 폐기했는지 기록해야 다시 만들지 않음

작은 버그 수정·문구 변경·스타일 조정은 ADR이 아닌 일반 커밋 메시지로 충분.

## 파일명 규칙

```
ADR-YYYYMMDD-짧은-제목.md
```

예시:
- `ADR-20260515-multi-persona-setup.md`
- `ADR-20260612-lawsuit-status-sheet-column.md`
- `ADR-20260718-rejected-react-migration.md`

날짜는 결정일(작성일이 아님). 같은 날 2개면 뒤에 `-2` 추가.

## 템플릿

새 ADR을 만들 때 아래 템플릿을 복사해 채워주세요.

```markdown
# ADR-YYYYMMDD-제목

## 상태
- 제안 / 채택 / 폐기 / 대체됨(by ADR-YYYYMMDD-XXX)

## 컨텍스트
- 이 결정이 필요해진 배경
- 관련 제약·요구사항·이해관계자
- 멀티 페르소나 모드였다면 어떤 직군이 어떤 의견을 냈는지

## 결정
- 무엇을 하기로 했는가 (한 문단)
- 핵심 변경점 bullet

## 근거
- 왜 이 결정인가
- 어떤 trade-off를 받아들였는가
- 데이터·실측·인터뷰 등 객관적 근거

## 대안 (검토했으나 채택하지 않은 것)
- 대안 1: 무엇이었고 왜 안 골랐는가
- 대안 2: 무엇이었고 왜 안 골랐는가

## 영향
- 코드: 어떤 파일·모듈이 영향받는가
- 사용자: 매니저 4명에게 어떤 변화가 보이는가
- 운영: GAS 배포·시트 마이그레이션 필요한가
- 후속 작업: 이 결정 다음에 해야 할 것들
- 리스크: 만약 결정이 잘못됐다면 어떤 신호로 알게 되는가

## 참조
- 관련 커밋 SHA
- 관련 ADR
- 외부 문서 URL
```

## 워크플로우

1. 결정 시점에 즉시 작성 (사후 작성하면 합리화로 변질)
2. 멀티 페르소나 모드 종료 직후 → 충돌 해결 결과를 ADR로
3. ADR 채택 후 코드 변경 시 커밋 메시지에 ADR 파일명 참조
4. 결정이 6개월 이내 뒤집어졌다면 새 ADR로 대체하고 원본은 "대체됨" 표시 (삭제 X)

## 절대 금지

- 작성된 ADR 삭제 금지 (틀린 결정도 학습 자료)
- 사후 수정으로 결정 미화 금지 — 새 ADR로 대체
- 개인정보·민감 정보 포함 금지
